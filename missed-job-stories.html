Missed Job Resolution

User stories for the flow that handles jobs that were never started — covering what the provider sees and does on mobile, and how the admin responds in the dashboard.

Backend endpoint live

Provider Mobile app What the assigned provider experiences before, during, and after a missed job
P–1

See that a past job needs attention

As a provider, I want to see a clear flag on any job whose date has passed without me clocking in, so I know I need to take action rather than just ignoring it.

Missed badge Amber banner

Acceptance criteria

  • Job detail header shows a red Missed status pill (replaces Upcoming / Confirmed)
  • Amber banner reads "This job was never started" with a Resolve button
  • Banner only appears for past-day jobs in committed statuses (assigned, confirmed, in_progress, on_the_way) that are not completed or cancelled
  • Completed and cancelled jobs never show the banner
P–2

Report that I actually worked the job

As a provider, I want to log my actual start and end times when I worked a job but forgot to clock in or out, so the office can verify my hours and I still get paid.

Resolve sheet Time steppers Note field

Acceptance criteria

  • Tapping Resolve opens a sheet with two paths: "I worked this job" and "I didn't make it"
  • Selecting "I worked this job" pushes a Log actual hours screen
  • Screen shows ± steppers for clock-in and clock-out times; hours cycle 1–12, minutes snap to :00/:15/:30/:45, AM/PM toggles independently
  • Times default to the booking's scheduled start and end
  • A required Note field prompts a brief description for the office to verify against
  • Submitting with end ≤ start shows "Invalid times — end time must be after start time"
  • Submitting with an empty note shows "Note required"
  • Valid submission sends PATCH /bookings/:id/resolve-missed with resolution: "worked", start/end ISO timestamps, and the note
  • Success alert: "Submitted — The office will review and update the job." then navigates back
P–3

Report that I couldn't make it

As a provider, I want to quickly tell the office why I didn't show up so they can handle the customer, without requiring me to make a phone call.

No-show reasons Reason required

Acceptance criteria

  • Selecting "I didn't make it" in the resolve sheet reveals a reason picker inline
  • Reasons: Sick or emergency · Vehicle / transport issue · Wrong location or locked out · Scheduling conflict · Other
  • Submit is disabled until a reason is selected
  • Tapping Submit sends PATCH /bookings/:id/resolve-missed with resolution: "no_show" and the selected reason
  • Success alert: "Reported — The office has been notified. They'll follow up if needed."
  • If the job was already resolved, server returns 409 and app shows an error toast — no crash
P–4

Clock out from an overdue job alert

As a provider who is still working past the scheduled end time, I want to clock out directly from the "Still working?" alert so I don't have to dismiss it and navigate back to the action bar.

Overdue timer Clock out

Acceptance criteria

  • "Still working?" alert fires every 30 min past the scheduled end time while the job is in_progress
  • Alert has two buttons: Still going (default) and Clock out (destructive)
  • Tapping Clock out triggers the same flow as the Clock Out button in the action bar — including the checklist-incomplete guard and the early-clockout reason picker if applicable
  • Tapping Still going dismisses the alert; it will fire again after the next 30-minute mark
Admin Dashboard What the office sees after a provider submits a resolution
⚠

The backend stores all resolution data and fires an in-app notification today. The dedicated admin review UI (stories A–2 through A–4) is not yet built — these stories define the intended experience for the next sprint.

A–1

Get notified when a provider resolves a missed job

As an admin, I want to receive an in-app notification the moment a provider submits a resolution so nothing falls through the cracks after the fact.

Shipped In-app notification

Acceptance criteria

  • On successful PATCH /bookings/:id/resolve-missed, the backend fires a booking.missed_resolution admin notification
  • Notification message: "Provider logged worked hours for booking #XXXX" or "Provider reported no-show for booking #XXXX"
  • Notification links to the booking detail page
  • Notification appears in the admin notifications panel without a page reload
A–2

Review a "worked" claim and approve or reject it

As an admin, I want to see the provider's submitted hours and note on the booking record so I can decide whether to mark the job complete and count it toward their pay.

Pending UI Next sprint

Acceptance criteria

  • Booking detail shows a "Missed — resolved by provider" section when missedResolution is present
  • Section displays: resolution type, submitted date/time, provider's claimed start and end times, and the note
  • Admin can tap Mark complete to transition the booking to completed and trigger the normal post-completion flow (receipt, review request)
  • Admin can tap Dispute to flag the claim for follow-up (sends a message to the provider thread)
  • Once actioned, the section shows who reviewed it and when
A–3

Handle a no-show and decide next steps

As an admin, I want to see the provider's reason for not showing up so I can contact the customer, reassign or cancel the job, and decide whether to log the provider no-show.

Pending UI Next sprint

Acceptance criteria

  • No-show resolution displays the provider's selected reason prominently on the booking detail
  • Quick-action buttons on the card: Reassign (opens provider assignment flow) · Cancel booking · Contact customer
  • Actioning any button marks the resolution as handled
  • Customer is not automatically notified — admin confirms before any outbound message is sent
A–4

See all unresolved missed jobs in one place

As an admin, I want a filtered view of bookings that are past their scheduled date and still unresolved so I can proactively follow up without relying solely on notifications.

Pending UI Next sprint

Acceptance criteria

  • Bookings list supports a Missed filter tab (past-day, committed status, no missedResolution, not completed/cancelled)
  • Each row shows: booking date, customer, assigned provider, and how many days overdue
  • Rows where the provider has submitted a resolution are visually differentiated from rows with no submission at all
  • Count badge on the Missed tab updates in real time as providers submit resolutions or admins action them
